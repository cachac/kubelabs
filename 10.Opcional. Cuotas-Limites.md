# 10. Cuotas y Límites <!-- omit in toc -->

> [Documentación Limit Ranges](https://kubernetes.io/docs/concepts/policy/limit-range/)
> [Documentación Resource Quotas](https://kubernetes.io/docs/concepts/policy/resource-quotas/)
> [Prácticas recomendadas](https://cloud.google.com/blog/products/containers-kubernetes/kubernetes-best-practices-resource-requests-and-limits)

Este laboratorio se basa en las actividades del laboratorio 9.

## Requisitos previos
- Contar con un clúster de Kubernetes funcional y `kubectl` configurado.
- Contar con el namespace `dev` para realizar las pruebas (crear si no existe: `kubectl create ns dev`).

## Objetivo
- Definir y validar ResourceQuotas y LimitRanges en Kubernetes utilizando la interfaz de línea de comandos (CLI).

---

### 1. Establecer el namespace dev como predeterminado
```bash
kubens dev

# O bien:
kubectl config set-context --current --namespace=dev
```

### 2. Crear el archivo `quota.yaml`
Crear el archivo `quota.yaml` con la definición de la cuota para el namespace `dev`:
```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: dev-quota
  namespace: dev
spec:
  hard:
    pods: "10"
    # request: recursos mínimos garantizados para el servicio
    requests.cpu: "250m" # 250 milicores = 1/4 core
    requests.memory: 100Mi
    requests.storage: 1000Mi
    # limit: no debe ser menor al request
    limits.cpu: "500m" # 500 milicores = 1/2 core
    limits.memory: 500Mi

    # Recursos adicionales limitados
    persistentvolumeclaims: "4"
    replicationcontrollers: "20"
    secrets: "10"
    services: "10"
    services.loadbalancers: "2"
```

### 3. Aplicar la cuota
```bash
kubectl apply -f quota.yaml
```

### 4. Consultar los detalles de la cuota
```bash
kubectl describe quota dev-quota
```
> La columna **HARD** muestra los límites definidos.
> La columna **USED** muestra los recursos que están siendo consumidos actualmente.

---

### 5. Validar el funcionamiento de la cuota
Las cuotas de recursos limitan la cantidad de objetos y el consumo total de recursos de cómputo en un namespace. Para comprobarlo, crear el archivo `pod-quota-dev.yaml`:
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: errorapp
  namespace: dev
  labels:
    app: errorapp
spec:
  containers:
  - name: errorapp
    image: "nginx:latest"
    resources:
      limits:
        cpu: 250m
        memory: 500Mi
      requests:
        cpu: 250m
        memory: 500Mi
```

### 6. Intentar aplicar el Pod
```bash
kubectl apply -f pod-quota-dev.yaml
```
Confirmar que el comando falla con el siguiente mensaje de error (debido a que se superan los 100Mi de memoria máxima solicitada en la cuota):
> `Error from server (Forbidden): error when creating "pod-quota-dev.yaml": pods "errorapp" is forbidden: exceeded quota: dev-quota, requested: requests.memory=500Mi, used: requests.memory=0, limited: requests.memory=100Mi`

### 7. Corregir el Pod y volver a aplicar
Modificar el archivo `pod-quota-dev.yaml` para ajustar `requests.memory` a `50Mi` y `limits.memory` a `100Mi` (valores permitidos dentro del límite de la cuota):
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: errorapp
  namespace: dev
  labels:
    app: errorapp
spec:
  containers:
  - name: errorapp
    image: "nginx:latest"
    resources:
      limits:
        cpu: 250m
        memory: 100Mi
      requests:
        cpu: 250m
        memory: 50Mi
```
Aplicar el archivo corregido:
```bash
kubectl apply -f pod-quota-dev.yaml
```

### 8. Verificar la actualización de la cuota
```bash
kubectl describe quota dev-quota
```
Verificar que la columna **USED** refleje el consumo de:
- `limits.cpu`
- `limits.memory`
- `pods`
- `requests.cpu`
- `requests.memory`

### 9. Eliminar los recursos creados
```bash
kubectl delete quota dev-quota -n dev
kubectl delete pod errorapp -n dev
```

---

### 10. Crear el archivo `limitrange.yaml`
Crear el archivo `limitrange.yaml` para definir límites de recursos por defecto en los contenedores del namespace `dev`:
```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: devlimits
  namespace: dev
spec:
  limits:
    - type: Container
      default: # Límite por defecto si no se especifica en el manifiesto
        cpu: 200m
        memory: 512Mi
      defaultRequest: # Solicitud por defecto si no se especifica en el manifiesto
        cpu: 100m
        memory: 256Mi
      max: # Límite máximo permitido
        cpu: 500m
        memory: 1024Mi
      min: # Solicitud mínima requerida
        cpu: 100m
        memory: 100Mi
```

### 11. Aplicar el LimitRange
```bash
kubectl apply -f limitrange.yaml -n dev
```

### 12. Describir el LimitRange
```bash
kubectl describe limitrange devlimits -n dev
```

### 13. Crear un Pod sin límites explícitos
Ejecutar un Pod sencillo:
```bash
kubectl run new-dev --image=nginx -n dev
```
Inspeccionar el Pod creado:
```bash
kubectl describe pod new-dev -n dev
```
Verificar que la sección `Limits` y `Requests` contenga los valores asignados automáticamente por el LimitRange:
```yaml
    Limits:
      cpu:     200m
      memory:  512Mi
    Requests:
      cpu:     100m
      memory:  256Mi
```

### 14. Probar la restricción del límite mínimo
Intentar aplicar el archivo `pod-quota-dev.yaml` original o modificado con `requests.memory: 50Mi` (el cual es menor al mínimo de `100Mi` requerido por el LimitRange):
```bash
kubectl apply -f pod-quota-dev.yaml
```
Confirmar que la creación es rechazada con el siguiente error:
> `Error from server (Forbidden): error when creating "pod-quota-dev.yaml": pods "errorapp" is forbidden: minimum memory usage per Container is 100Mi, but request is 50Mi`

### 15. Ajustar el archivo y volver a aplicar
Modificar el archivo `pod-quota-dev.yaml` asignando `100Mi` en `requests.memory` para cumplir con el límite mínimo:
```yaml
      requests:
        cpu: 250m
        memory: 100Mi
```
Aplicar el archivo ajustado:
```bash
kubectl apply -f pod-quota-dev.yaml
```

### 16. Eliminar el LimitRange
```bash
kubectl delete limitrange devlimits -n dev
```

### 17. Limpiar el entorno
Eliminar los recursos creados en el namespace `dev`:
```bash
kubectl delete all --all -n dev
```
Restaurar el namespace predeterminado del contexto:
```bash
kubens default

# O bien:
kubectl config set-context --current --namespace=default
```
